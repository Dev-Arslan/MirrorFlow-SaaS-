Order flow

Three roles, one execution pipeline

Every order a Master places passes through the same signed, whitelisted pipeline before it's resized and routed to each connected Follower account.

01 — Master

Trade on your own account

Connect a standard Binance Futures API key, scoped to Futures trading only. Trade exactly as you normally would — no dashboard to learn.

02 — Engine

Signed, whitelisted relay

Orders are captured via authenticated WebSocket, validated, and resized against each Follower's equity and leverage caps before routing.

03 — Followers

Positions mirror automatically

Entries, additions, and closes replicate on each Follower's own account, using their own funds and their own pre-set risk limits.

Key handling

Your API key never leaves the whitelist

MirrorFlow only ever requests trade-scoped permissions. Withdrawals stay impossible at the exchange level, not just by policy.

IP whitelist enforced per key

Each API key is locked to MirrorFlow's static execution IPs at the Binance account level — requests from anywhere else are rejected before they reach your account.

Withdrawal permission disabled

Keys are requested with Futures trading enabled and withdrawals off — the setting Binance enforces, not a promise MirrorFlow makes.

Per-follower risk ceilings

Max leverage, max position size, and daily loss caps are set by each Follower and enforced before any mirrored order is placed.

One-click disconnect

Revoking the key on Binance or unlinking inside MirrorFlow halts mirroring for that account instantly.

Built for both sides

Everything the trade relay needs to handle

Sizing, leverage, drawdown — the details that break naive copy bots are handled at the engine level.

Proportional sizing

Follower orders scale to their own equity, not the Master's notional — a 0.1 BTC master trade can mirror as 0.005 BTC.

Multi-master routing

Followers can allocate across several Masters at once, with per-master weight and exposure caps.

Sub-second reconciliation

Performance ledger

Circuit breakers

Latency dashboard

Master revenue share

Masters earn a configurable performance fee on profitable Follower trades, settled automatically each cycle.

Sizing logic

Three ways to mirror a trade

Followers choose how a Master's order translates onto their own account — this setting can be changed anytime without affecting open positions.

Mode A

Exact copy

Follower opens the identical quantity and leverage as the Master. Best when Follower equity closely matches the Master's.

Mode B

Percentage copy

Order size scales to a fixed % of Follower equity relative to the Master's, so risk stays proportional regardless of account size.

Mode C

Fixed USDT copy

Every mirrored entry uses a flat USDT notional set by the Follower, independent of what the Master risked on that trade.
